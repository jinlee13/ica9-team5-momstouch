## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **서비스명** | 똑똑한 엄마 (가칭) |
| **한 줄 정의** | 아이 생년월일만 입력하면 발달 단계에 맞는 육아용품을 즉시 추천하고, 쿠팡 파트너스를 통해 바로 구매까지 연결하는 웹 서비스 |
| **서비스 유형** | 웹 기반 육아용품 추천 + 구매 타이밍 엔진 + 상시 딜 구매 채널 |
| **목표 시장** | 국내 0~36개월 영유아 부모 |

### 서비스 배경

한국 베이비 시장은 약 5조 원 규모이며, 프리미엄 제품 비중이 60% 이상을 차지하고 연 6.7% 성장 중이다. 초보 부모들은 아이에게 무엇을 언제 사줘야 하는지 몰라 SNS, 인플루언서, 육아 카페 등 파편화된 정보를 수시간씩 탐색하고 있다. 이 서비스는 그 탐색 과정을 없애고 **"지금 이 아이에게 딱 맞는 것"을 즉시 제시**하는 것을 목표로 한다.

## 2. 문제 정의 & 기회

### 현재 초보 부모가 겪는 문제

| # | 문제 | 결과 |
|---|------|------|
| 1 | 개월 수별로 무엇이 필요한지 모름 | 필요한 것을 빠뜨리거나 너무 늦게 구매 |
| 2 | 정보가 너무 많고 출처가 파편화되어 있음 | 탐색 피로 → 잘못된 결정 |
| 3 | 필요 없는 제품까지 구매 | 예산 낭비 + 후회 |
| 4 | 인플루언서 공동구매 타이밍 놓침 | 더 비싼 가격에 구매 |

### 시장 기회

- 연간 출생아 수 약 24만 명 (2024년), 지속적인 신규 유저 유입 기반 존재
- 글로벌 베이비 시장 2026년 $2,700~3,760억 달러, CAGR 5~7%
- 한국 베이비 시장 CAGR 6.7% — 저출산에도 **1인당 지출 증가(골드키즈 트렌드)**
- 기존 경쟁사들은 "정보 제공"에 집중 → **"구매 의사결정 지원"이라는 공백 존재**

## 3. 타겟 사용자 & 페르소나

### 주요 타겟

- 25~40세 초보 부모 (0~36개월 자녀를 둔 부모)
- 스마트폰 사용에 익숙하나 육아 정보 탐색에 지쳐 있음

> **임산부는 MVP 타겟 제외.** 서비스 입력값이 "아이 생년월일"이므로 출산 전 사용 불가. 출산예정일 기반 사전 준비 모드는 Phase 2 로드맵에서 별도 검토 예정.

### 페르소나 A — "당황한 초보 엄마" 지은 씨 (32세)

> "아이를 낳고 보니 뭐가 필요한지 아무것도 모르겠어요. 퇴원하고 집에 오자마자 검색부터 시작했는데, 정보가 너무 많아서 오히려 더 혼란스러워요."

- **상황:** 출산 직후 (0~1개월), 첫째 아이
- **니즈:** 지금 당장 필요한 것과 곧 필요할 것을 한눈에 파악하고 싶음
- **불만:** 너무 많은 정보, 신뢰할 수 없는 출처, 광고성 추천

### 페르소나 B — "바쁜 워킹맘" 미래 씨 (35세)

> "아이가 6개월인데, 이유식 준비를 해야 한다고 하는데 뭐가 필요한지 모르겠어요. 검색할 시간도 없어요."

- **상황:** 복직 후 육아·일 병행 중
- **니즈:** 빠르게 핵심만 파악, 구매 링크까지 바로 연결
- **불만:** 탐색에 쓸 시간이 없음

## 4. 핵심 가치 제안 & 차별점

### 핵심 가치 제안

> **"지금 우리 아이에게 딱 필요한 것만, 딱 맞는 타이밍에"**

### 경쟁사 대비 차별점

| 서비스 | 포지셔닝 | 한계 |
|--------|----------|------|
| 마미톡 | 육아 정보 피드 | 구매 의사결정 연결 약함 |
| 베이비빌리 | 생애주기 구조 | 추천 깊이 부족 |
| 맘가이드 | 안전성 분석 | 범위가 좁음 |
| 맘맘 | 국민템 랭킹 | 개월 수 맞춤 없음 |
| **똑똑한 엄마** | **구매 타이밍 엔진** | — |

### 와우 포인트 2가지

1. **인플루언서 없이도 OK** — 수십 개의 SNS와 카페를 돌아다니지 않아도 신뢰할 수 있는 추천을 즉시 제공 → 시간의 획기적인 절약
2. **발달이론 기반 타이밍** — 단순 나이가 아닌 아이의 발달 단계에 맞춘 구매 시점 안내 → 미리 준비 가능, 다른 정보 탐색 불필요

### 발달이론과 추천의 연결 원리

> **"왜 지금 이게 필요한지"**를 발달 근거로 설명하는 것이 이 서비스의 핵심 신뢰 자산.

| 발달 단계 | 개월 수 | 발달 이정표 | 연결 추천 아이템 |
|---------|--------|-----------|--------------|
| 신생아기 | 0-3M | 청각 자극 반응, 수유 리듬 형성 | 속싸개, 신생아 바운서, 수유쿠션 |
| 목 가누기 | 3-6M | 고개 들기, 손 뻗기 시작 | 바운서→범보의자 전환, 딸랑이 |
| 이유식 시작 | 6-12M | 앉기 시작, 고형식 소화 가능 | 이유식 의자, 이유식기, 방수턱받이 |
| 첫걸음 준비 | 12-18M | 서기·걷기 시도, 탐색 욕구 폭발 | 보행기/보행보조기, 안전문 |
| 활동기 | 18-24M | 달리기, 계단 오르기, 소근육 발달 | 킥보드, 그림책, 블록 |
| 언어폭발기 | 24-36M | 언어 발달 급속, 자아 주장 | 그림책, 역할 놀이 완구 |

→ 각 상품 상세 페이지에서 이 발달 근거 문구를 **"왜 지금?"** 형태로 노출

## 5. 서비스 기능 정의 (MVP)

### v1 Scope Freeze — 이번 배포에서 확정된 범위

> GPT 검토 반영: MVP 범위를 명시적으로 잠금. 아래 표 기준으로 개발 범위 논쟁 종결.

| 기능 | v1 포함 | 근거 |
|------|:------:|------|
| 온보딩 (생년월일 입력) | ✅ | 핵심 입력 |
| 개월 수 자동 계산 (클라이언트) | ✅ | 핵심 로직 |
| 추천 목록 (지금/곧/아직 탭) | ✅ | 핵심 가치 |
| 카테고리 필터 (4개 상위) | ✅ | MVP 범위 |
| 상품 상세 — 정적 정보 + 구매 링크 | ✅ | 구매 연결 |
| 체크리스트 (로컬스토리지) | ✅ | 핵심 UX |
| 가성비 딜 배너 (쿠팡 파트너스 수동) | ✅ | 수익화 |
| 실시간 가격 조회 (네이버 API) | ❌ Phase 2 | 외부 의존성 高 |
| KC/리콜 자동 연동 | ❌ Phase 3 | 외부 의존성 高 |
| 인기지수 자동화 (쇼핑인사이트) | ❌ Phase 3 | 외부 의존성 高 |
| GPT 추천 이유 자동 생성 | ❌ Phase 2 | v1은 수작업 seed |
| 10개 세부 카테고리 전체 | ❌ Phase 2 | 데이터 입력량 과다 |
| 회원가입/로그인 | ❌ Phase 2 | scope 외 |
| 협업 필터링/AI 고도화 | ❌ Phase 3 | 데이터 필요 |

### MVP 범위 요약

- **추천 구간:** 6구간 (0-3M / 3-6M / 6-12M / 12-18M / 18-24M / 24-36M)
- **v1 카테고리:** 4개 상위 카테고리 (상위만 필터, 세부 확장은 Phase 2)
- **추천 항목:** 구간 × 카테고리 핵심 3개 = **총 72개 항목** → seed 데이터로 직접 입력 필요 (코드보다 데이터가 먼저)

**카테고리 구조**

| 상위 카테고리 (v1) | 세부 카테고리 (Phase 2) |
|----------------|----------------------|
| 먹기 | 수유, 이유식·식기 |
| 자기·위생 | 수면, 위생·기저귀, 목욕·스킨케어 |
| 놀기·배우기 | 놀이·발달 |
| 외출·안전 | 외출, 안전, 건강관리, 가구·생활 |

### 기능 목록

#### F-01. 온보딩 — 아이 정보 입력

- 아이 생년월일 입력 (날짜 선택 UI)
- 현재 개월 수 자동 계산
- 첫 방문 시 온보딩 설명 화면 (2~3 슬라이드)

#### F-02. 홈 대시보드 — 지금/곧/아직 구분

- **지금 필요:** 현재 개월 수에 해당하는 필수 아이템
- **곧 필요:** 향후 1~2개월 내 필요 예상 아이템 (미리 준비 유도)
- **아직 이른 것:** 3개월 이후 필요한 아이템 (과잉구매 방지)

#### F-03. 카테고리 탐색

- 10개 핵심 카테고리별 필터 탐색
- 각 아이템에 추천 이유 텍스트 제공
- **필수도 분류 배지:** 필수 / 상황형 / 생략가능 / 중고·대여 권장

#### F-04. 상품 상세 페이지 + 구매 채널 연결

> **팀 피드백 반영:** 결제서비스(구매 경로)를 명확히 보여준다.

- **"왜 지금?" 발달 근거 문구** — 이 시기에 왜 이 제품이 필요한지 1~2줄로 표시
- 가격대 범위 안내 (정적, v1 기준: 수작업 입력)
- 안전 인증 여부 — v1은 수동 입력값 표시 (자동 연동은 Phase 3)
- **쿠팡 파트너스 "바로 구매" 버튼** — 가장 눈에 띄는 위치, 새 탭으로 이동
- 광고/제휴 표시 배지 (법적 의무)
- 체크리스트 상태 변경 버튼 (구매완료/보류/생략)

#### F-05. 체크리스트

- 아이템별 상태 저장: **구매완료 / 보류 / 생략**
- 진행률 표시 (예: "이번 달 필수템 5개 중 3개 완료")
- MVP는 로컬스토리지 저장 → Phase 2에서 계정 연동

#### F-06. 상시 딜 가격 구매 채널

> **팀 피드백 반영:** "공구가로 상시 구매할 수 있는 사이트"로 포지셔닝. 특정 시간에만 열리는 인플루언서 공구가 아닌, **언제든 좋은 가격에 바로 살 수 있는 채널** 제공.

- 현재 개월 수 연관 추천 상품 배너 (쿠팡 파트너스 링크 — v1은 수동 입력)
- "지금 살 수 있어요" 메시지와 함께 노출 — 시간 제한 없는 상시 접근 가능 강조
- 광고/제휴 배지 표시 (법적 의무)
- ※ 인플루언서 연계 그룹 바잉(공동구매)은 Phase 3 이후 별도 검토

### 세부 카테고리 10개

수유 / 수면 / 위생·기저귀 / 목욕·스킨케어 / 외출 / 안전 / 놀이·발달 / 이유식·식기 / 건강관리 / 가구·생활

### MVP 제외 기능 (Phase 2 이후)

- 회원가입/로그인
- 다자녀 프로필 관리
- 커뮤니티/댓글
- 푸시 알림
- iOS/Android 네이티브 앱

| 체크리스트 버튼 | 구매완료 / 보류 / 생략 | 로컬스토리지 |

### 화면 5 — 체크리스트

| 요소 | 설명 |
|------|------|
| 진행률 바 | "이번 달 필수템 5개 중 3개 완료" |
| 탭 필터 | 전체 / 구매완료 / 보류 / 생략 |

## 9. 비기능 요구사항

### 성능

- 홈 대시보드 초기 로딩: **3초 이내**
- 추천 목록 응답: **1초 이내**
- **모바일 퍼스트** 디자인 (90% 이상 모바일 예상 트래픽)

### 접근성 & 호환성

- iOS Safari, Android Chrome 최신 2개 버전 지원

### 보안

- 생년월일 데이터: MVP에서 로컬스토리지 저장 (서버 전송 없음)
- 외부 API 키: 서버사이드에서만 호출
- HTTPS 필수

### 법적 고려사항

- **개인정보보호법:** MVP에서 생년월일을 서버에 저장하지 않으므로 수집 최소화 원칙 준수
- **크롤링:** 외부 사이트 robots.txt 준수
- **광고 표시:** 쿠팡 파트너스 링크에 "광고/제휴" 표시 의무
- **면책 문구:** "참고용 정보이며 의료적 조언이 아닙니다" 표시
