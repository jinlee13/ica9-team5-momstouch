## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **서비스명** | 똑똑한 엄마 |
| **한 줄 정의** | 아이 생년월일만 입력하면 발달 단계에 맞는 육아용품을 즉시 추천하고, 사이트 내 장바구니에서 바로 구매할 수 있는 웹 서비스 |
| **서비스 유형** | 웹 기반 육아용품 추천 + 구매 타이밍 엔진 + 자체 쇼핑 채널 |
| **목표 시장** | 국내 0~36개월 영유아 부모 |
| **배포 URL** | https://ddokddok-mom.vercel.app |
| **GitHub** | https://github.com/jinlee13/ica9-team5-momstouch |

### 서비스 배경

한국 베이비 시장은 약 5조 원 규모이며, 프리미엄 제품 비중이 60% 이상을 차지하고 연 6.7% 성장 중이다. 초보 부모들은 아이에게 무엇을 언제 사줘야 하는지 몰라 SNS, 인플루언서, 육아 카페 등 파편화된 정보를 수시간씩 탐색하고 있다. 이 서비스는 그 탐색 과정을 없애고 **"지금 이 아이에게 딱 맞는 것"을 즉시 제시**하는 것을 목표로 한다.

## 2. 문제 정의 & 기회

### 현재 초보 부모가 겪는 문제

| # | 문제 | 결과 |
|---|------|------|
| 1 | 개월 수별로 무엇이 필요한지 모름 | 필요한 것을 빠뜨리거나 너무 늦게 구매 |
| 2 | 정보가 너무 많고 출처가 파편화되어 있음 | 탐색 피로 → 잘못된 결정 |
| 3 | 필요 없는 제품까지 구매 | 예산 낭비 + 후회 |
| 4 | 인플루언서 공동구매 타이밍 놓침 | 더 비싼 가격에 구매 |

### 시장 기회

- 연간 출생아 수 약 24만 명 (2024년), 지속적인 신규 유저 유입 기반 존재
- 글로벌 베이비 시장 2026년 $2,700~3,760억 달러, CAGR 5~7%
- 한국 베이비 시장 CAGR 6.7% — 저출산에도 **1인당 지출 증가(골드키즈 트렌드)**
- 기존 경쟁사들은 "정보 제공"에 집중 → **"구매 의사결정 지원"이라는 공백 존재**

## 3. 타겟 사용자 & 페르소나

### 주요 타겟

- 25~40세 초보 부모 (0~36개월 자녀를 둔 부모)
- 스마트폰 사용에 익숙하나 육아 정보 탐색에 지쳐 있음

> **임산부는 MVP 타겟 제외.** 서비스 입력값이 "아이 생년월일"이므로 출산 전 사용 불가. 출산예정일 기반 사전 준비 모드는 Phase 2 로드맵에서 별도 검토 예정.

### 페르소나 A — "당황한 초보 엄마" 지은 씨 (32세)

> "아이를 낳고 보니 뭐가 필요한지 아무것도 모르겠어요. 퇴원하고 집에 오자마자 검색부터 시작했는데, 정보가 너무 많아서 오히려 더 혼란스러워요."

- **상황:** 출산 직후 (0~1개월), 첫째 아이
- **니즈:** 지금 당장 필요한 것과 곧 필요할 것을 한눈에 파악하고 싶음
- **불만:** 너무 많은 정보, 신뢰할 수 없는 출처, 광고성 추천

### 페르소나 B — "바쁜 워킹맘" 미래 씨 (35세)

> "아이가 6개월인데, 이유식 준비를 해야 한다고 하는데 뭐가 필요한지 모르겠어요. 검색할 시간도 없어요."

- **상황:** 복직 후 육아·일 병행 중
- **니즈:** 빠르게 핵심만 파악, 장바구니에 담아 바로 구매
- **불만:** 탐색에 쓸 시간이 없음

## 4. 핵심 가치 제안 & 차별점

### 핵심 가치 제안

> **"지금 우리 아이에게 딱 필요한 것만, 딱 맞는 타이밍에"**

### 경쟁사 대비 차별점

| 서비스 | 포지셔닝 | 한계 |
|--------|----------|------|
| 마미톡 | 육아 정보 피드 | 구매 의사결정 연결 약함 |
| 베이비빌리 | 생애주기 구조 | 추천 깊이 부족 |
| 맘가이드 | 안전성 분석 | 범위가 좁음 |
| 맘맘 | 국민템 랭킹 | 개월 수 맞춤 없음 |
| **똑똑한 엄마** | **구매 타이밍 엔진 + 자체 구매 채널** | — |

### 와우 포인트

1. **인플루언서 없이도 OK** — 수십 개의 SNS와 카페를 돌아다니지 않아도 신뢰할 수 있는 추천을 즉시 제공
2. **발달이론 기반 타이밍** — 단순 나이가 아닌 아이의 발달 단계에 맞춘 구매 시점 안내
3. **자체 장바구니 + 결제 흐름** — 외부 사이트로 이동 없이 사이트 내에서 바로 구매 가능

### 발달이론과 추천의 연결 원리

| 발달 단계 | 개월 수 | 발달 이정표 | 연결 추천 아이템 |
|---------|--------|-----------|--------------|
| 신생아기 | 0-3M | 청각 자극 반응, 수유 리듬 형성 | 속싸개, 신생아 바운서, 수유쿠션 |
| 목 가누기 | 3-6M | 고개 들기, 손 뻗기 시작 | 바운서→범보의자 전환, 딸랑이 |
| 이유식 시작 | 6-12M | 앉기 시작, 고형식 소화 가능 | 이유식 의자, 이유식기, 방수턱받이 |
| 첫걸음 준비 | 12-18M | 서기·걷기 시도, 탐색 욕구 폭발 | 보행기/보행보조기, 안전문 |
| 활동기 | 18-24M | 달리기, 계단 오르기, 소근육 발달 | 킥보드, 그림책, 블록 |
| 언어폭발기 | 24-36M | 언어 발달 급속, 자아 주장 | 그림책, 역할 놀이 완구 |

## 5. 서비스 기능 정의 (MVP — 배포 완료)

### 구현 완료 기능

| 기능 | 상태 | 비고 |
|------|:----:|------|
| 온보딩 (생년월일 입력) | ✅ 완료 | localStorage 저장, 개월 수 자동 계산 |
| 홈 대시보드 (지금/곧/아직 탭) | ✅ 완료 | 4개 카테고리 필터 포함 |
| **연령 기반 상품 필터링** | ✅ 완료 | 생년월일 → 개월 수 → 추천 상품만 표시 |
| 상품 상세 페이지 | ✅ 완료 | 발달 근거 문구 + 시중 상품 카드 |
| 시중 상품 목록 (market_products) | ✅ 완료 | 5,049개 크롤링 상품 |
| 시중 상품 상세 페이지 (/market/[id]) | ✅ 완료 | 대형 이미지, 상세이미지, 상품정보표 |
| **전체 상품 카탈로그 (/browse)** | ✅ 완료 | 기저귀·물티슈 등 모든 카테고리 직접 탐색 |
| **장바구니 (/cart)** | ✅ 완료 | localStorage 기반, 수량 조절, 합계 |
| **결제 페이지 (/checkout)** | ✅ 완료 | 배송지 입력, 결제수단 선택, 주문확인 (UI) |
| 체크리스트 (/checklist) | ✅ 완료 | localStorage 기반, 진행률 표시 |
| GNB 장바구니 뱃지 | ✅ 완료 | 실시간 담긴 수량 표시 |
| 생년월일 변경 안전처리 | ✅ 완료 | confirm() 다이얼로그 후 초기화 |
| 데이터 없는 제품 숨김처리 | ✅ 완료 | market_products 매핑된 제품만 홈에 노출 |

### 보류/Phase 2 기능

| 기능 | 상태 | 이유 |
|------|:----:|------|
| 실제 결제 PG 연동 | ⏸ 보류 | 사업자등록 + PG사 계약 필요 |
| LLM 챗봇 (Claude API) | ⏸ 보류 | API 사용 비용 발생 |
| YouTube 영상 추천 | ⏸ 보류 | API 사용 비용 발생 |
| 회원가입/로그인 | ❌ Phase 2 | scope 외 |
| 실시간 가격 조회 (네이버 API) | ❌ Phase 2 | 외부 의존성 高 |
| 푸시 알림 | ❌ Phase 2 | 앱 아님 |
| KC/리콜 자동 연동 | ❌ Phase 3 | 외부 의존성 高 |

### 카테고리 구조 (현재 구현)

| 대분류 | 소분류 |
|--------|--------|
| 먹기 | 젖병, 수유, 이유식, 식기 |
| 자기·위생 | 수면, 기저귀, 물티슈, 세제·세탁, 위생용품 |
| 놀기·배우기 | 장난감, 완구, 교육 |
| 외출·안전 | 유모차, 카시트, 외출용품 |

## 6. 사용자 여정 (User Flow)

```
[서비스 진입 — ddokddok-mom.vercel.app]
           │
           ▼
[온보딩] 아이 생년월일 입력 → localStorage 저장
           │
           ▼
[홈 대시보드 /home]
  지금 필요 🔥 | 곧 필요 ⏰ | 아직 이른 것 📦
  카테고리 필터: 전체 / 자기·위생 / 먹기 / 놀기·배우기 / 외출·안전
           │
     ┌─────┴─────┐
     ▼           ▼
[추천 제품 클릭]  [전체 상품 탐색 /browse]
[/products/[id]]  기저귀, 물티슈 등 5,049개 상품
     │
     ▼
[시중 상품 카드] → [상품 상세 /market/[id]]
     │                대형 이미지 + 상세이미지
     ▼
[담기 버튼] → 장바구니 추가 (GNB 뱃지 업데이트)
     │
     ▼
[장바구니 /cart]
  수량 조절 / 삭제 / 합계 확인
     │
     ▼
[결제 /checkout]
  배송지 입력 → 결제 수단 선택 → 주문완료
     │
     ▼
[체크리스트 /checklist]
  구매완료 / 보류 / 생략 상태 관리
```

## 7. 주요 화면 정의

### 화면 1 — 온보딩 (/)

| 요소 | 설명 |
|------|------|
| 서비스 로고 & 한 줄 소개 | "우리 아이 개월 수에 딱 맞는 육아용품 추천" |
| 날짜 입력 컴포넌트 | 생년월일 선택 |
| CTA 버튼 | "추천 받기" |
| 개인정보 안내 | "생년월일은 기기에만 저장됩니다" |

### 화면 2 — 홈 대시보드 (/home)

| 요소 | 설명 |
|------|------|
| GNB | 로고 / 전체상품 / 체크리스트 / 🛒뱃지 / 생년월일변경 |
| 아이 개월 수 카드 | "지금 우리 아이는 7개월이에요" + DDOK 발달 나침반 |
| 진행률 카드 | "이번 달 필수템 N개 중 N개 완료" |
| 곧 필요 배너 | 앞으로 2개월 내 필요한 제품 미리 안내 |
| 우선순위 탭 | 지금 필요 🔥 / 곧 필요 ⏰ / 아직 이른 것 📦 |
| 카테고리 필터 | 사이드바(PC) / 수평 스크롤(모바일) |
| 전체 상품 배너 | "/browse 전체 상품 둘러보기" CTA |

### 화면 3 — 추천 상품 상세 (/products/[id])

| 요소 | 설명 |
|------|------|
| 발달 근거 문구 | "왜 지금?" — 이 시기에 이 제품이 필요한 이유 |
| 시중 상품 카드 | 썸네일(80px), 브랜드, 상품명, 가격, 평점 |
| 담기 버튼 | 장바구니 추가 + 확인 표시 |
| 카드 클릭 | /market/[id] 상세 페이지로 이동 |

### 화면 4 — 시중 상품 상세 (/market/[id])

| 요소 | 설명 |
|------|------|
| 메인 이미지 | 정사각형 대형 이미지 |
| 썸네일 갤러리 | 최대 10장, 클릭 시 메인 전환 |
| 상세이미지 | 전체 스크롤 상세 이미지 (detail_images) |
| 상품 정보표 | 브랜드, 가격, 평점, 리뷰 수, 권장연령 |
| 장바구니 담기 | 수량 선택 후 담기, 성공 확인 |

### 화면 5 — 전체 상품 카탈로그 (/browse) ← 신규

| 요소 | 설명 |
|------|------|
| 대분류 탭 | 자기·위생 / 먹기 / 놀기·배우기 / 외출·안전 |
| 소분류 필터 | 기저귀, 물티슈, 젖병, 이유식기 등 버튼 |
| 5열 상품 그리드 | 이미지 + 브랜드 + 상품명 + 가격 + 평점 |
| 더 보기 | 페이지네이션 (20개씩) |
| 각 상품 | /market/[id] 링크 + 담기 버튼 |

### 화면 6 — 장바구니 (/cart)

| 요소 | 설명 |
|------|------|
| 상품 목록 | 썸네일 + 이름 + 가격 + 수량 +/- + 삭제 |
| 합계 | 상품 소계 + 배송비(5,000원) + 총액 |
| 결제하기 버튼 | /checkout 이동 |
| 빈 장바구니 상태 | "쇼핑하러 가기" CTA |

### 화면 7 — 결제 (/checkout)

| 요소 | 설명 |
|------|------|
| 배송지 입력 | 이름, 연락처, 주소 |
| 결제 수단 | 신용카드 / 계좌이체 / 카카오페이 (UI만) |
| 주문 요약 | 상품 목록 + 최종 금액 |
| 주문완료 | 주문번호 발급, 주문 내역 확인 안내 |

> ⚠️ **현재 결제는 UI만 구현.** 실제 PG 연동은 사업자등록 및 PG사 계약 후 Phase 2에서 진행.

### 화면 8 — 체크리스트 (/checklist)

| 요소 | 설명 |

## 9. 비기능 요구사항

### 성능

- 홈 대시보드 초기 로딩: **3초 이내**
- 추천 목록 응답: **1초 이내**
- **모바일 퍼스트** 디자인

### 접근성 & 호환성

- iOS Safari, Android Chrome 최신 2개 버전 지원

### 보안

- 생년월일 데이터: localStorage 저장 (서버 전송 없음)
- Supabase API 키: 서버사이드 환경변수로만 관리 (NEXT_PUBLIC_ 접두사 주의)
- HTTPS 필수 (Vercel 자동 적용)

### 법적 고려사항

- **개인정보보호법:** MVP에서 생년월일을 서버에 저장하지 않으므로 수집 최소화 원칙 준수
- **크롤링:** 외부 사이트 robots.txt 준수
- **면책 문구:** "참고용 정보이며 의료적 조언이 아닙니다" 표시
